Ejercicio

Las bases de datos relacionales (SQL, mariadb - NoSQL, mongodb) se basan en organización de la información mediante pequeñas partes que se integran mediante “Identificadores” ¿Categorías? El identificador es aquello capaz de relacionar dos o más conjuntos de datos. Usualmente las tablas son bases de datos relacionales y los documentos (JSON) son bases de datos no relacionales. (Al parecer, un identificador es una estructura de datos, un lenguaje común para acceder a los distintos conjuntos de datos), En cambio, en una base de datos no relacional, cada conjunto de datos podría tener su propia estructura. Por ejemplo, en una ciudad está el conjunto de personas (Cada persona es análogo a un conjunto de datos) que hablan cierto idioma. Si todos hablaran el mismo idioma podríamos acceder a sus conocimientos con el mismo lenguaje. Si hay distintos conjuntos de la sociedad que hablan un idioma distinto, entonces distintas personas tienen distintos identificadores o estructuras con las cuales se puede acceder a su información. Por ejemplo, si una serie de datos estuviera organizada en partes por filas, un identificador podría ser el número de fila.
